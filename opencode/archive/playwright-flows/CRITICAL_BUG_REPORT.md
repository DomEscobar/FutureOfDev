## Summary

During full flow testing (Fighter Creation → Match Join → Battle), the backend throws a database error preventing matches from completing.

## Error Details

```
ERROR: insert or update on table "match_score_fighters" 
violates foreign key constraint "match_score_fighters_fighter_id_fkey"
(SQLSTATE 23503)
```

## Root Cause Analysis

The newly created fighter's ID is not being properly inserted into the `match_score_fighters` table when the battle begins. This suggests:

1. Fighter ID mismatch between `fighters` table and `match_score_fighters` table
2. Or fighter not being properly committed before match starts
3. Or async timing issue (fighter creation not completed before match join)
